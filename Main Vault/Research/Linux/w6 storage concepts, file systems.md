RAID Video
- RAID handles data loss prevention
- Redundant Array of Independent Disks
- The data is copied onto multiple disks. In the event of a disk failure, nothing would be lost. 

| RAID Level                                | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ----------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RAID 0 (striping)                         | A stripe set breaks data into units and stores the units across a series of disks by reading and writing to all disks simultaneously. Striping:<br><br>- Provides an increase in performance.<br>- Does not provide fault tolerance. A failure of one disk in the set means all data is lost.<br>- Requires a minimum of two disks.<br>- Has no overhead because all disk space is available for storing data.                                                                                                                                                                                                                                                                                                                                                                                                                          |
| RAID 1 (mirroring)                        | A mirrored volume stores data to two (or more) duplicate disks simultaneously. If one disk fails, data is present on another disk. The system switches immediately from the failed disk to a functioning disk. Mirroring:<br><br>- Provides fault tolerance for a single disk failure.<br>- Does not increase performance.<br>- Requires a minimum of two disks.<br>- Has overhead. Overhead is 1 / n where n is the number of disks. If data is written twice, half of the disk space is used to store the second copy of the data.<br>- RAID 1 is the most expensive fault tolerant system.                                                                                                                                                                                                                                           |
| RAID 5 (striping with distributed parity) | A RAID 5 volume combines disk striping across multiple disks with parity for data redundancy. Parity information is stored on each disk. If a single disk fails, its data can be recovered using the parity information stored on the remaining disks. Striping with distributed parity:<br><br>- Provides fault tolerance for a single disk failure.<br>- Provides an increase in performance for read operations. Write operations are slower with RAID 5 than with other RAID configurations because of the time required to compute and write the parity information.<br>- Requires a minimum of three disks.<br>- Has an overhead of one disk in the set for parity information (1 / n).<br>    - A set with 3 disks has 33% overhead.<br>    - A set with 4 disks has 25% overhead.<br>    - A set with 5 disks has 20% overhead. |
| RAID 10 (stripe of mirrors)               | A RAID 10 volume stripes data across mirrored pairs and across multiple disks for data redundancy. If a single disk fails, its data can be recovered using the mirrored information stored on the remaining disks. If two disks in the same mirrored pair fail, all data will be lost because there is no redundancy in the striped sets. RAID 10:<br><br>- Provides fault tolerance for a single disk failure.<br>- Provides redundancy and performance.<br>- Uses 50% of the total raw capacity of the drives due to mirroring.<br>- Requires a minimum of four disks.                                                                                                                                                                                                                                                                |

File System Types
- ext2
	- Second extended file system
	- 1993
	- Stores data in a heirarchical fashion
	- Maximum file size of 2TB
	- Max volume size of 4TB
	- File names go up to 255 characters
	- Takes a very long time to recover after an abrupt shutdown
	- e2fsck
		- Ext2 File System Check
		- Fixed problems from unexpected shutdowns
		- Saves lost data to lost+found
		- Takes a long time
		- Analyzes the entire system
		- Can take 10 minutes to several hours
- ext3
	- Third extended file system
	- Does everything ext2 does
	- Includes a journal
	- Journals rectifies unclean system shutdown
	- Marks transactions as pending / incomplete
	- They can then be fixed / completed after the unexpected shutdown
	- Eliminates need to check the entire system
	- The journaling process uses more system memory
- ext4
	- Maximum vol size: 1 exabyte
	- Max file size: 16 TB
	- Max num files: 4 billion
	- Max file name length: 255 characters
	- Uses a journal with checksums to prevent corruption and make it more reliable
- btrfs
	- Uses copy on write
		- Provides storage pools and snapshots
	- Once you've allocated storage devices to pool, you can mount them at various points
	- Snapshots allow you to restore lost files very quickly
- xfs
	- Fast and portable
	- Ported to Linux

|Scheduler|Description|
|---|---|
|Noop|Noop is the simplest scheduler. It places all I/O requests into a First in/First Out (FIFO) queue. In addition, read/write requests of a similar purpose are also combined to reduce the number of disk operations and increase the length of system calls. This scheduler is often used for systems that do not need an I/O scheduler. For example, when a virtual machine (VM) is running on a host computer that's already using its own I/O scheduler.|
|Deadline|The Deadline scheduler creates a read queue and a write queue. Since each I/O request has an associated timestamp (used by the kernel for an expiration time), the Deadline scheduler utilizes this timestamp to push I/O requests that've reached their deadline to their highest priority.<br><br>The default Deadline values are 500 ms for read operations and 5,000 ms for write operations. If needed, these values can be adjusted. Because of these values, the Deadline scheduler is often considered the optimal scheduler for read-heavy workloads.|
|CFQ|The Complete Fairness Queuing (CFQ) input/output (I/O) scheduler works by creating a per-process I/O queue.<br><br>The goal of CFQ is to provide a fair I/O priority to each process. This is accomplished by first ordering the queues to reduce disk seeking and then servicing these per-process I/O queues, in a round-robin fashion. The benefits of using the CFQ scheduler is that it tries to provide each process with the same priority for disk access. The disadvantage is that it makes this scheduler less optimal for environments that might need to prioritize one request type (such as reads) from a single process.|
