### Directories
**Dir Navigation**
- pwd
	- Present working directory
	- Displays the current directory
	- Tilde represents the /home directory of the current user
- cd
	- Change directory
	- By itself, returns you to home
	- Relative Path
		- Specify a subdirectory of the current directory
		- Path is relative to some other point in the file system
		- Example (while in /home/rtracy) temp
		- .. moves you to a higher directory
		- ../.. moves you two spaces up
	- Absolute Path
		- Absolute paths use the full path
		- Starts with the root directory, which is '/'
		- Example: /home/rtracy/temp
- ls
	- ls by itself lists the contents of the current directory
	- If you enter a path (relative or absolute) to a directory, ls will list its contents
	- Includes useful options
		- -a shows all files, including hidden ones
		- -l shows all files in a long list
		- -al does both
	- If you want to sort the results, use the --sort option
		- ls -l --sort=size
		- You can also sort by time, version, and extension
- tree
	- A utility that shows the files in the directory in a tree
	- tree by itself shows the current directory and all subdirectories
	- tree -a shows everything
	- tree -f prints the relative path along with the file names
	- tree -d only shows directories

- Create Directory
	- mkdir
		- mkdir followed by the name of the directory creates that directory in your current path
		- you can specify the path that you'd like to use by doing `mkdir /path` 
		- -p will create all of the directories in the path if they don't already exist
- Copy / Move / Delete
	- cp
		- Copies directories
		- `cp /path1 /path2`
		- `cp -R` copies recursively; it includes everything in the copy, including subdirectories
	- mv
		- Moves directories
			- mv copies the directory and deletes the source directory
			- `mv /path1 /path2`
		- Can be used to rename a directory

- Delete Directory
	- rmdir
		- Removes directory
		- `rmdir /path`
		- Will only delete if empty unless you use the rm -r command
	- rm
		- rm -r recursively deletes everything
		- `rm -r /path`

### Links
|Type|Description|
|---|---|
|Hard link|A hard link is a duplicate entry in the file system that points to a specific piece of data on the disk drive. With a hard link:<br><br>- The same inode is used as the original file. The inode specifies where a file's data physically exists on a disk.<br>    <br>    The **ls -i** command displays the inodes for the files and directories in a directory.<br>    <br>- No data is stored in a hard link. The data is pointed to by the inode, which the hard link refers to.<br>- Files are basically hard links when created, pointing to the inode where the data is stored. Until all hard links that point to an inode and its data are deleted, the data will not be removed.<br>- In the output from the **ls -a** command, a hyphen is used as the first character in the permission string for a hard link, which is the same character used for normal files (for example, **-** rwxr-xr-x).|
|Symbolic link|A symbolic link (also known as a soft link) is a file that points to another file in the file system. A symbolic link is similar to shortcuts in the Windows OS. With a symbolic link:<br><br>- Separate inodes are used. The link file has an inode that is distinct from the inode of the file being pointed to.<br>- In the output from the **ls -l** command:<br>    - A lower-case L (l) is used as the first character in the permission string (for example, lrwxrwxrwx indicates a symbolic link).<br>    - The -> character sequence follows the file name, which is followed by the file that the link points to.|

Links
- Link files direct you to something else
- Hard Link File stores to the Inode of a file
	- Two files shares the same Inode
	- You can't tell which one is the pointer or the pointee
	- To the Linux system, they're the same file
	- Treated as a single file
- Symbolic Link
	- Has its own Inode
	- Pointer and pointee can be easily distinguished
	- Remain unique files
	- If you modify the pointer, the pointee is modified
- Use the ls -l command to see links
- Create links with the ln command
	- `ln /pointee_path /pointer_path`
		- `ln -s` creates symbolic
		- `ln` creates hard
	- Pointer comes second
	- You're saying create a link that points to this from this
	- Links don't contain anything. Everything is contained by the target
- ls -l info
	- first character indicates type of file
		- d = dir
		- - = normal / hard-link
		- l = symbolic link
			- Also has an -> to show where it points to
- Renaming or moving symbolic linked files can create issues
- You can't create hard links to directories or on different drives

### Filesystem Hierarchy Standard (FHS)
FHS
- Provides a standard directory structure across all Linux distros
- /bin
	- Executable files necessary to manage and run the Linux system
	- cp and rm
- /boot
	- Contains bootloader files for system bootloading
- /dev
	- Unique. Contains hardware devices
	- Each hard disk is called SDA, SDA1, SDA2...
	- Contains files to manage hardware
	- Character-oriented device files: for devices that send or receive data, like printer, mouse, or tape drive
	- Block-oriented device files: for devices that manage data in blocks, like HD and Flash Drive.  Permits random access.
	- /dev contains config for
		- Floppy drive : fd0
		- Optical drive : sr0
		- Serial port : ttyS0
		- Parallel port : lp0
		- Hard Disks : sda, sdb, sdc
		- Partitions : sda1, sda2
		- Random number : random
- /etc
	- Text based config files for the Linux system and all the applications running on it
- /home
	- Contains sub directories for each user account
- /media
	- Used by some to mount external devices
- /mnt
	- To temporarily mount local and remote file systems
- /opt
	- Contains files for some applications installed on the system
- /proc
	- Unique
	- Doesn't actually exist
	- Pseudo file system
	- Dynamically created when accessed
	- Several files in here are numbers that correspond to running processes. They're called PIDs
- /root
	- Root users home directory
- /sbin
	- Contains system admin utilities
- /srv
	- Contains subdirectories where services running on the system save files
- /tmp
	- Contains temporary files created by the system
	- Clears older temporary files on a periodic basis
- /usr
	- Contains application files
- /var
	- Contains variable data, such as log files

### Locating and Searching Files
**FIND**
- `find /search_dir -name name`
- `find / -user username`
- `find / -size size`
- Searches Linux filesystem for a file
- By default, begins in the directory you start in and then each subdirectory
- If the file name contains spaces, contain it with quotes in the command

**LOCATE**
- Similar to find
- Produces results faster
- Builds and searches an index instead of manually searching directories
- You need to install it to use it
- Install it using RPM, dpkg, zypper, YUM/dnf
- Create an index
	- stored in /bar/something
- Update the database
	- Happens daily by default
	- updatedb will do it too and can take a long time
	- sudo updatedb
- `locate file_name`
- Can be edited in the /etc/updatedb.conf

**WHICH**
- Used to display the full path to a shell command or utility
- Finds executable file for a command
- Type `which ls` to find the path to the `ls` command
- Only looks in directories in the $PATH directories

**WHEREIS**
- Used to find a file for commands
- Displays more information than WHICH
- Also displays man page location, and where the command source code is.
- Type `whereis ls` to find the executable and man pages for `ls`

**TYPE**
- Determines if its a alias, function, keyword, or built-in command.
- `type ln` returns information about `ln`
- If a command is hashed, it has been run recently

- grep can be used for a text string in a file or in files
- `grep term /target`
- Searches the target file for the term
- grep -i
	- Ignores case
- grep -l
	- Displays only the name of the file where the text was found

	- Displays matching line numbers

	- Searches recursively

- metacharacters
	- \* any number of characters

	- \ are used to escape out of these metacharacters
	- man regex shows all the other metacharacters you can use
- fgrep
	- Fixed string grep
	- Direct string comparisons
	- `fgrep pattern /path`
	- Same as grep -F

### Text Stream Processing
Text Stream
- Any information redirected from the standard output of a command to the standard input of another command
- You can intercept and process the text stream with the following commands

Command Overviews
- sed
	- Capture a text stream and modify it
	- `sed -i` edits the file in place
	- `sed -i s` substitutes instead of deleting
	- `sed -i s/old/new`
	- `sed -i s/old/new/g` does a global check
- awk
	- More powerful variant of sed
	- Defines text files as databases
	- Each record is a new line
	- Each field is $1, $2, $3 and are separated by spaces
	- `cat customers | awk '{print $2}'` prints the second field of the record customers
	- `cat customers | awk '{print "First: "$1"\t", "Last: "$2"\t", "Phone: "$3""}'`
- tr
	- Translate or delete characters
	- `cat customers | tr a-z A-Z`
		- Translates all lowercase to uppercase

Words are delimited by white space.<br>- **-s** specifies the number of initial characters to skip.<br>- **-w** specifies the number of characters to compare in each line.<br>- **-u** leaves out the duplicate lines.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      | **uniq myfile** omits all repeated lines in myfile. It prints the first occurrence only.  <br>  <br>**uniq -d myfile** prints only the repeated lines.  <br>  <br>**uniq -u myfile** prints only the unique lines.  <br>  <br>**uniq -f 4 myfile** skips the first four words when comparing lines.  <br>  <br>**uniq -s 4 myfile** skips the first four characters when comparing lines.  <br>  <br>**uniq -w 4 myfile** uses only the first four characters when comparing lines.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **wc**       | Prints the number of bytes, characters, lines, or words, or the length of the longest line from the text of a file or text stream. Options include the following:<br><br>- **-c** specifies bytes.<br>- **-m** specifies characters. Character count is often identical to byte count.<br>- **-l** specifies line count.<br>- **-L** specifies the length of the longest line.<br>- **-w** specifies word count.<br><br>When no options are used, the command prints line count, word count, and byte count, respectively.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               | **wc myfile** displays line, word, and character count.  <br>  <br>**wc -L myfile** displays the length of the longest line in the file.  <br>  <br>**wc -m myfile** displays the number of characters in the file.
