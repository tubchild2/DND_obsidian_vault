| Term                   | Definition                                                                                                                                                                    |
| ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Standard user          | A user account that can log in to the system.                                                                                                                                 |
| System or service user | An account that's created by default during the Linux installation and used by the system for specific roles.                                                                 |
| Primary group          | A group created by default on most Linux distributions when a standard user is created (also called a private group). They're used to manage access to files and directories. |
| Secondary group        | A group type that manages access to files and directories, similar to a primary group. Users often belong to multiple secondary groups.                                       |
| Shadow file            | A file (such as the /etc/shadow file) that holds passwords and password expiration information for user accounts. The /etc/gshadow file holds passwords for groups.           |

### Users
- You can create several user accounts with unique configuration
	- You need to enter username and password usually. 
	- Others use pluggable authentication modules (PAM)
	- Authentication is stored in 
		- /etc/passwd
			- Also stores daemons
			- User information lines are split by colons
				- Username
				- Password (legacy)
				- UID
				- GID
					- Group ID
					- Primary group of the user
				- Full Name / Description
				- Home Directory
				- Default Shell
					- /bin/bash
					- /sbin/nologin
		- /etc/shadow
			- Information for each user including password
			- Works with passwd to store passwords
				- Should they become unsynchronized, things could go very wrong
				- `pwck` utility verifies links in the two files
				- `pwconv` fixes sync errors
			- Each pass is contained on a single colon-split line
				- Username
				- Password
					- If it's blank, it has no password
					- ! means locked and can't log in
					- \* means something but I missed it
				- Last Mod
					- Last time the password has changed
				- Min Days
					- Minimum number of days that must elapse before the password can be changed
				- Max Days
					- Maximum number of days before a user must change his password
					- Typically set to 999999 days
				- Days Warning
					- Number of days before a password is set to expire or warned to change their password
				- Disabled Days
					- Usually unpopulated
					- Represents number of days after the password expires to destroy the account
				- Expire
					- Usually unpopulated
		- /etc/group
			- Referenced in passwd
- Once logged in, your user environment is created
- `finger -l` gives you information
- id username gives returns your unique user id (uid)
	- Users are assigned uids 1000, 1001, 1002... 
	- The root user usually has a uid of 0
	- 0-499 are reserved for system administration

### Groups
- Grouped to organize user permission categories
- Users inherit group permissions
- Saves time so that permissions don't need to be assigned to each user individually
- Groups make Linux systems scalable
- Group IDs start at 1000 (usually)
- /etc/group
	- Stores all the groups in the system
	- Groupname:Password:GID:User1,User2,User3
- /etc/gshadow
	- Groupname:Password:Group_Admins:Group_Members

|Type|Description|
|---|---|
|Standard user|Standard user accounts can log in to the system. Standard user accounts:<br><br>- Generally have easy-to-remember usernames (such as Mary or bkaun). An administrator must create the usernames.<br>- Have an ID of 500 or more (on some distributions) or 1,000 or more (on other distributions). The ID is automatically assigned by the system when the account is created.|
|System or service accounts|System user accounts (also called service user accounts) are created by default during the Linux installation and are used by the system for specific roles.<br><br>System user accounts:<br><br>- Have names that correspond with their roles, such as ftp and mail.<br>- Can't be used to log in to the system.<br>- Have an ID of 500 or less (on some distributions) or 1,000 or less (on other distributions). The ID is automatically assigned by the system when the account is created.<br><br>The root user account is created by default and has a UID of 0. However, the root user account can be used to log in to the system and perform tasks.|
|Primary group|Primary groups (also called the private group) are created by default on most Linux distributions when a standard user is created. These accounts help manage access to files and directories. Primary groups:<br><br>- Have the corresponding user as the only member.<br>- Are automatically assigned as the owner of files and directories when they are created in the file system.<br>- Are similar to any other group (the only difference is that the group is identified as the primary group in the user account's configuration).|
|Secondary group|Secondary groups are also used to manage access to files and directories. Secondary groups:<br><br>- Are not automatically assigned user accounts as members.<br>- Receive their membership as assigned by the system administrator.|

### Group Management

`groupadd options groupname`
- -g new group ID
- -p new group password
- -r creates system group

usermod is used for adding people to groups
- usermod -g
	- sets the users default group
	- `usermod -g users kmorgan`
- usermod -G 
	- sets the user's additional groups
	- erases all current additional group memberships and replaces them with whatever you put in here
	- `usermod -G users, RandD kmorgan`

groupmod is available on some Linux distros
- `groupmod options groupname`
	- -g changes groupid
	- -p changes group password
	- -A adds a user account to the group
		- `groupmod -A "username" groupname`
	- -R removes a user account from membership in the group
	- -n renames

groupdel deletes a group
- `groupdel groupname`
